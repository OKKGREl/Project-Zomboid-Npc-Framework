title=PZNS NPC Framework
description=[h1]PZNS Framework (B41)[/h1]
description=
description=A new Lua based framework for managing, spawning, and directing NPCs in Project Zomboid
description=
description=This is an effort to standardize NPC management and spawning while offering the flexibility to all modders the ability to create and modify NPCs to their content without worry about all the complicated issues such as AI behaviors, data management, and decompiling PZ source code.
description=
description=[h1]THIS IS A PREVIEW WITH B41[/h1]
description=- [i][b]There are no Multiplayer Support planned[/b][/i]
description=- Basically what I have learned to manage PZ game data.
description=- No promises on anything; if you want something, please contribute to the framework github.
description=- Join the Modding Community Discord to follow developments
description=- With the debugmode active (via sandbox-options), you can spawn up to 2 NPCs using the contextmenu [b]PZNS Debug World[/b]
description=

description=[h1]There are no Multiplayer Support planned[/h1]
description=- Technical reasons, in short:
description=[list]
description=[*]TIS devs already have NPCs on their roadmap, this is a mod to get NPCs sooner for personal enjoyment.
description=[*]I am not doubling/tripling my own workload to write server-side code and testing all that stuff.
description=[*]If anyone else wants to try - go ahead, I am not stopping anyone from trying, the project's code is all there on github.
description=[/list]
description=Cows: [i][b]I will be blocking anyone requesting multiplayer support from me personally.[/b][/i]
description=
description=[h1]This mod is INCOMPATIBLE with SS/SSC[/h1]
description=- In theory, it [i]should[/i] be mostly compatible, but...
description=- For stability reasons (mostly OnRenderTicks cycles) and how poorly SS/SSC manages game data, I am declaring PZNS as incompatible.
description=- If you want to break your game with SS/SSC, I am not responsible for your choice.
description=
